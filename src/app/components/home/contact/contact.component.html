Building a Serverless Portfolio

Building a Serverless Portfolio with Angular, AWS, and DynamoDB

Introduction

In today's fast-paced world, a well-crafted portfolio is crucial for showcasing your skills and projects. This blog explores the technical architecture behind a serverless portfolio, leveraging Angular for the frontend and various AWS services for the backend.

Frontend: Angular

The frontend of the portfolio is built using Angular, a powerful and robust framework for creating dynamic web applications. Angular provides a modular structure, making it easier to organize and maintain code while ensuring a seamless user experience.

Backend: AWS Lambda and API Gateway

The backend is powered by AWS Lambda, a serverless compute service that allows you to run code without provisioning or managing servers. API Gateway is utilized to create RESTful APIs, acting as a bridge between the frontend and Lambda functions, facilitating smooth communication.

-

Hosting: S3 Bucket, CloudFront, and Route53

For hosting the frontend files, an Amazon S3 bucket is employed, providing a scalable and secure solution for static web hosting. CloudFront, a content delivery network (CDN), enhances performance by delivering content globally with low latency. Route53 handles domain management, ensuring a professional and personalized web address.

- -

Contact Form: DynamoDB

To enable user interaction, a contact form is incorporated, allowing visitors to get in touch seamlessly. The data submitted through the form is persisted in DynamoDB, a fully managed NoSQL database service by AWS. DynamoDB ensures reliable storage and retrieval of user-submitted data.

Conclusion

By combining Angular for the frontend, AWS Lambda and API Gateway for serverless backend, S3 Bucket and CloudFront for hosting, Route53 for domain management, and DynamoDB for data persistence, this portfolio achieves a scalable, performant, and cost-effective architecture. Embracing serverless technologies empowers developers to focus on building great applications without the hassle of managing infrastructure.
